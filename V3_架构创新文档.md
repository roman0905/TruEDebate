# TruEDebate (TED) 深度审查与架构创新文档

> **结论先行**：原论文实现存在 4 个关键问题，其中"MHA 退化"是阻碍指标提升的根本原因。
> 本次重构提出 3 大创新点 + 1 个鲁棒损失函数，预期能显著提升 F1(Fake) 指标。

---

## 一、深度审查发现的问题

### 🔴 问题 1: MHA 数学退化（最严重，影响巨大）

**论文公式 Eq.10**:
```
c = MHA(e_F^proj, g^proj, g^proj)
```
其中 `g = GlobalPool(H^(L))` 是图级别的**单一向量**。

**原代码实现**:
```python
q = e_proj.unsqueeze(1)     # [B, 1, P]  ← 序列长度 = 1
kv = g_proj.unsqueeze(1)    # [B, 1, P]  ← 序列长度 = 1
attn_output, _ = self.mha(q, kv, kv)
```

**数学分析**：
- Attention 权重 = `softmax(Q·K^T / √d)`
- 当 K 的序列长度为 1 时，Q·K^T 形状为 `[B, h, 1, 1]`
- 单元素的 softmax 永远等于 1.0
- 因此 `MHA(q, kv, kv) = Linear(V)` —— **完全失去注意力机制的意义**

这意味着论文描述的"News-Debate Interactive Attention"在严格按公式实现时**根本不存在**！这就是为什么无论怎么调超参，F1(Fake) 都难以突破 0.64。

---

### 🔴 问题 2: 数据分布严重不平衡且 Train ≠ Val/Test

论文 Table 2 数据：

| Split | Real | Fake | Real:Fake |
|-------|------|------|-----------|
| Train | 2,878 | 1,006 | **2.86:1** |
| Val   | 1,030 | 244   | **4.22:1** |
| Test  | 1,024 | 234   | **4.38:1** |

**问题分析**：
- 类别权重从训练集估计：`real_weight=0.67, fake_weight=1.93`
- 但 Val/Test 比训练集**更不平衡**，这种"先验偏移"使评估指标失真
- F1(Fake) 在 234 个样本上波动剧烈：每错分 1 个样本影响 ~0.4%

**现有方案不足**：
- 单纯的 class weight 对所有 fake 样本一视同仁，无法聚焦于"难样本"
- Label smoothing 0.05~0.1 缓解作用有限

---

### 🔴 问题 3: 辩论的二分图结构未被利用

**论文核心洞察**："Truth becomes clearer through debate" —— Pro vs Opp 的对抗是关键。

**原代码实现**:
```python
graph_repr = global_mean_pool(x, batch)  # 平均所有 7 个节点
```

**问题**：
- 把 Proponent 节点（0, 2, 4）和 Opponent 节点（1, 3, 5）混在一起平均
- **完全丢失立场信息**
- 如果 Pro 阵营论据强 vs Opp 阵营论据强，平均后无法区分

---

### 🔴 问题 4: 网络架构其他瑕疵

1. **GATConv 表达力不足**：使用原始 GAT，建议升级为 GATv2Conv
2. **残差连接缺失**：GAT 第一层无残差，深层网络易梯度消失
3. **分类器过简**：单层 Linear，不利于学习复杂的特征交互

---

## 二、创新方案：V3 架构设计

### 创新点 1: Node-Level Cross-Attention（修复 MHA 退化）

**改进思路**：让 News 真正"注意"到每个辩论节点，而不是单一聚合向量。

**实现**：
```python
# 节点级投影：[total_nodes, gat_hidden] → [total_nodes, P]
node_proj = self.node_proj(x)

# 转换为稠密批：[total_nodes, P] → [B, max_nodes, P]
node_dense, node_mask = to_dense_batch(node_proj, batch)

# Cross-Attention: News 作为 Query，所有节点作为 K/V
q = e_proj.unsqueeze(1)                        # [B, 1, P]
attn_output, _ = self.cross_attn(
    q, node_dense, node_dense,                  # ← K/V 是真正的多个节点
    key_padding_mask=~node_mask,
)
c_attn = attn_output.squeeze(1) + e_proj      # 残差连接
```

**理论意义**：
- 现在 K 的序列长度 = 7（6 辩论节点 + 1 Synthesis）
- Attention softmax 对 7 个节点产生不同权重
- News 可以**有选择地关注**最相关的辩论论据
- 这才是"Interactive Attention"的真正含义

---

### 创新点 2: Stance-Aware Dual-Branch Pooling（立场感知双分支）

**改进思路**：分别聚合正反方节点，显式建模立场分歧。

**实现**：
```python
# Pro 节点掩码 (roles 0, 2, 4)
pro_mask = (role_dense == 0) | (role_dense == 2) | (role_dense == 4)
# Opp 节点掩码 (roles 1, 3, 5)
opp_mask = (role_dense == 1) | (role_dense == 3) | (role_dense == 5)

# 分别聚合
g_pro = masked_mean_pool(node_dense, pro_mask)  # [B, P]
g_opp = masked_mean_pool(node_dense, opp_mask)  # [B, P]

# 立场分歧信号（关键创新）
divergence = torch.abs(g_pro - g_opp)  # [B, P]
```

**理论意义**：
- `divergence` 大 → 双方阵营论据差异显著 → 辩论激烈，新闻更可能存在争议
- `divergence` 小 → 双方观点接近 → 论据可能被某一方主导
- 这是对"truth becomes clearer through debate"的**显式量化**

---

### 创新点 3: Multi-View Feature Fusion（多视图融合分类器）

**最终分类器输入**：
```
combined = [g_global; c_attn; divergence]  # 3 * proj_dim
```

三个视图各司其职：
| 视图 | 含义 | 来源 |
|------|------|------|
| `g_global` | 全图整体语义 | 论文原始路径 |
| `c_attn` | 新闻-辩论细粒度交互 | 创新点 1 |
| `divergence` | 立场分歧信号 | 创新点 2 |

**分类器架构**：
```python
nn.Sequential(
    nn.Linear(3*P, P),       # 特征融合
    nn.LayerNorm(P),         # 归一化
    nn.GELU(),               # 平滑激活
    nn.Dropout(),            # 正则化
    nn.Linear(P, 2),         # 分类输出
)
```

---

### 创新点 4: Focal Loss for Class Imbalance（焦点损失）

**公式**：
```
FL(p_t) = -α_t · (1 - p_t)^γ · log(p_t)
```

其中 `(1 - p_t)^γ` 是焦点项：
- 模型对样本预测越准确（p_t 越大），权重越小
- 模型预测越错误（p_t 越小），权重越大
- **自动聚焦于难样本**，比 class weight 更鲁棒

**实现要点**：
- `gamma=2.0`（论文推荐值）
- 兼容 `alpha`（class weight）
- 兼容 `label_smoothing`

---

### 辅助改进

1. **GATv2Conv 替换 GATConv**：注意力机制更具表达力（动态注意力）
2. **GAT 残差连接**：第一层加 `gat_input_proj` 对齐维度，第二层加普通残差
3. **更深的分类器**：从单层 Linear 升级为 2 层 + LayerNorm + GELU

## 六、未来研究方向

如果 V3 仍未达到论文指标，可考虑：

### 方向 A: 数据层面
- **检查辩论数据质量**：人工抽查 50 条 fake 样本的辩论是否合理
- **样本平衡采样**：使用 `WeightedRandomSampler` 让每个 batch 更平衡
- **数据增强**：随机 dropout 部分辩论节点作为正则

### 方向 B: 架构层面
- **Heterogeneous GAT**：为 Pro/Opp/Synthesis 节点使用不同的注意力头
- **Edge attributes**：为不同类型的边（时序/对抗/综合）学习不同权重
- **Synthesis 特殊处理**：将 Synthesis 节点单独编码后与图表示对比

### 方向 C: 训练策略
- **两阶段训练**：先冻结 BERT 训练 10 epoch，再联合微调
- **Adversarial training**：FGSM/PGD 对 BERT embedding 加扰动
- **Self-distillation**：用 best model 作为 teacher 蒸馏

### 方向 D: 损失函数
- **Asymmetric Loss**：对正负样本使用不同的 gamma
- **Dice Loss**：直接优化 F1 分数
- **Polynomial Loss**：在 CE 基础上加多项式正则

## 附：与原论文实现的对比表

| 模块 | 原论文实现 | V3 实现 | 改进点 |
|------|-----------|---------|--------|
| GAT | GATConv | GATv2Conv + 残差 | 表达力 ↑ |
| 池化 | global_mean_pool | mean + Pro/Opp 双分支 | 利用结构信息 |
| MHA | 单向量 K/V (退化) | 节点级 K/V | 真正实现注意力 |
| 分类器 | Linear(2P, 2) | Sequential(3P→P→2) | 容量 ↑ |
| 损失 | CrossEntropy + class_weight | Focal Loss + alpha | 难样本聚焦 |
| 特征 | [g_proj; c] (2P) | [g_global; c_attn; divergence] (3P) | 立场显式建模 |

**总结**：V3 不是简单的超参数调整，而是针对论文实现中**数学退化、结构信息丢失、类别不平衡**三大根本问题的系统性重构。
