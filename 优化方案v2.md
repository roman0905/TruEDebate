# TED 优化方案 V2

## 问题诊断

经过三轮训练实验，发现：

| 训练 | LR | 最佳Epoch | Val macF1 | Test macF1 | 问题 |
|------|-----|-----------|-----------|------------|------|
| 第一次 | 1e-5 | 12 | 0.7795 | 0.7779 | 学习率过低，过拟合严重 |
| 第二次 | 1e-4 | 6 | 0.7790 | 0.7708 | 较稳定，但容量不足 |
| 第三次 | 2e-4 | 3 | 0.7730 | 0.7674 | 学习率过高，Epoch 4崩溃 |

**核心发现**：
1. 与论文差距稳定在 3-5%，超参数微调无法解决
2. 模型容量可能不足（GAT_HIDDEN_DIM=128 太小）
3. 过拟合严重（Train Loss 降至 0.18，Val Loss 升至 0.55+）

## V2 优化策略

### 1. 增加模型容量
```python
GAT_HIDDEN_DIM = 128 → 256  # 提升 GAT 表达能力
PROJ_DIM = 128 → 256        # 与 GAT 维度保持一致
```

**理由**：论文可能使用了更大的隐藏层，当前 128 维可能无法充分捕获辩论图的复杂语义。

### 2. 回归论文原始 Dropout
```python
GAT_DROPOUT = 0.3 → 0.1
CLASSIFIER_DROPOUT = 0.3 → 0.1
```

**理由**：增加模型容量后，需要降低正则化强度，让模型充分学习。

### 3. 稳定学习率 + 更长 Warmup
```python
LEARNING_RATE = 2e-4 → 1e-4  # 回退到稳定值
WARMUP_RATIO = 0.1 → 0.15    # 更平滑的学习率上升
EARLY_STOPPING_PATIENCE = 7 → 8
```

**理由**：2e-4 导致训练不稳定（Epoch 4 崩溃），1e-4 是经过验证的稳定值。

### 4. 增强 GAT 残差连接
```python
# 在 GAT 第二层添加残差连接
if identity is not None and identity.shape == x.shape:
    x = x + identity
```

**理由**：残差连接缓解梯度消失，帮助深层网络训练。

### 5. 改进图池化策略
```python
# 使用 mean + max pooling 组合
graph_mean = global_mean_pool(x, batch)
graph_max = global_max_pool(x, batch)
graph_repr = graph_mean + graph_max
```

**理由**：
- Mean pooling 捕获平均语义
- Max pooling 捕获显著特征
- 组合使用增强图表示的丰富性

### 6. 保持温和的类别权重
```python
# 使用平方根逆频率（已实现）
weights = sqrt(total / count) 归一化
# real=0.74, fake=1.26（相比之前的 0.67/1.93 更温和）
```

## 预期效果

基于容量提升和更稳定的训练：
- **Test Accuracy**: 0.87-0.88
- **Test Macro F1**: 0.78-0.80
- **F1(Fake)**: 0.64-0.66

## 如果仍无改善

需要考虑更深层的问题：

1. **数据质量**：
   - 检查生成的辩论数据是否合理
   - Synthesis 节点内容是否有效
   - 是否需要数据增强

2. **架构根本性差异**：
   - 论文可能使用了 3 层 GAT（而非 2 层）
   - 可能使用了不同的 BERT 微调策略
   - 可能使用了集成学习

3. **训练策略**：
   - 尝试两阶段训练：先冻结 BERT，再联合微调
   - 尝试不同的 BERT 冻结层数（0/6/8/10）
   - 尝试更大的 batch size（8 或 16）

## 下一步实验

3. 如果 F1(Fake) < 0.64，调整类别权重为线性逆频率
